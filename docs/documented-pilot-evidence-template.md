# Documented Pilot Evidence Template

Use this only after a real file owner has asked for a documented pilot. This is a capture packet for turning one useful cleanup into anonymized proof later, not a substitute for a customer quote or buyer outcome.

## Private by default

Do not commit or publish:

- Company names.
- Supplier, farmer, cooperative, buyer, or importer names.
- Source rows, raw files, shipment references, receipt emails, or buyer documents.
- Farm coordinates, polygon files, or row-level geolocation data.

Publish only with explicit written approval.

## Public Fields, If Approved

- File type.
- Row count.
- Issue counts and blocker categories.
- Cleaned-pack contents at a category level.
- Issues fixed.
- Remaining source-owner follow-ups.
- Turnaround time.
- Buyer result, if the customer approves sharing it without names.
- One approved quote without naming the company.

## Capture Checklist

Before cleanup:

- File count.
- Row count.
- Commodity and source country.
- Buyer/importer deadline.
- Blocker count and warning count.
- Point-only over-4ha count.
- Missing plot or farm ID count.
- Missing supplier identity count.
- Duplicate farm ID count.
- Invalid coordinate row count.

After cleanup:

- Cleaned-pack contents returned.
- Issues fixed.
- Issues still blocked by missing source-owner data.
- Buyer/supplier follow-ups requested.
- Turnaround time.
- Buyer result, if known.

Quote permission:

- Approved short quote.
- Whether TraceReady may publish the quote without naming the company.
- Approved role label, if publishable.

Customer approval:

- May publish anonymized issue counts: yes/no.
- May publish before/after structure without names or coordinates: yes/no.
- May publish cleaned-pack contents at a category level: yes/no.
- May mention buyer/importer outcome without names: yes/no.
- Approval date and approver role.
